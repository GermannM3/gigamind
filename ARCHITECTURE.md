# 🏗️ Архитектура проекта GigaMind

```
┌─────────────────────────────────────────────────────────────────┐
│                    🧠 GigaMind Architecture                     │
└─────────────────────────────────────────────────────────────────┘

                    ┌─────────────────────┐
                    │   🧍 Пользователь   │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   🌐 Gradio UI      │
                    │  (Веб-интерфейс)    │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   🤖 main.py        │
                    │ (Основная логика)   │
                    └─────┬──────────┬────┘
                          │          │
        ┌─────────────────▼─┐      ┌─▼─────────────────┐
        │   🧠 memory.py    │      │   ⚖️ judge.py     │
        │ (Система памяти)  │      │ (Самооценка)      │
        └─────┬─────────────┘      └───────────────────┘
              │
   ┌──────────▼──────────┐
   │  🗄️ SQLite Database  │
   │  (Хранение истории) │
   └─────────────────────┘

                    ┌─────────────────────┐
                    │   ☁️ GigaChat API   │
                    │  (Генерация ответов)│
                    └─────────────────────┘


┌─────────────────────────────────────────────────────────────────┐
│                        Поток данных                             │
└─────────────────────────────────────────────────────────────────┘

1. Пользователь вводит запрос через Gradio интерфейс
2. Запрос передается в main.py
3. main.py обращается к memory.py для поиска контекста
4. memory.py извлекает релевантные данные из SQLite
5. main.py формирует промпт с контекстом
6. Запрос отправляется в GigaChat API для генерации ответа
7. Ответ сохраняется в memory.py (SQLite)
8. judge.py оценивает качество ответа
9. Результат с оценкой возвращается пользователю через Gradio


┌─────────────────────────────────────────────────────────────────┐
│                     Компоненты системы                          │
└─────────────────────────────────────────────────────────────────┘

📁 Файлы проекта:
├── main.py          # Оркестратор всех компонентов
├── memory.py        # Система памяти (RAG + хранение)
├── judge.py         # Система самооценки
├── .env             # API ключи и настройки
└── requirements.txt # Зависимости

💾 Хранилища:
├── SQLite           # Долговременная память
└── FAISS индексы    # Векторный поиск по истории

📡 Внешние API:
└── GigaChat API     # Генерация текста

🖥️ Интерфейсы:
└── Gradio           # Веб-интерфейс

🧠 Модели:
├── sentence-transformers # Эмбеддинги для поиска
└── VADER Sentiment       # Анализ тональности

┌─────────────────────────────────────────────────────────────────┐
│                   Преимущества архитектуры                      │
└─────────────────────────────────────────────────────────────────┘

✅ Модульность - каждый компонент может быть заменен или улучшен
✅ Легковесность - оптимизировано для работы на слабом железе
✅ Расширяемость - легко добавлять новые функции
✅ Безопасность - API ключи хранятся отдельно
✅ Надежность - есть обработка ошибок и логирование
```
